PAGE PRÉSENTATION PROJET S4 POLYTECH

1 - Présentation du projet

L’équipe a choisi de développer une application web de jeux cognitifs de croiser plusieurs centres d’intérêt : la programmation, le jeu vidéo et les neurosciences. Le produit vise à proposer des jeux courts, accessibles et progressifs permettant à l’utilisateur de stimuler certaines fonctions cognitives, comme la mémoire, l’attention et la rapidité de traitement.

L'application s'adresse à toute personne souhaitant "entraîner son cerveau", se divertir ou suivre sa progression grâce aux jeux "infinis"

2 - Objectifs du projet

2.1 - Objectif principal

Concevoir une plateforme web modulaire capable d'héberger plusieurs mini-jeux cognitifs, avec une première version fonctionnelle intégrant au minimum un jeu de type Memory.

2.2 - Objectifs secondaires

  • Proposer une interface claire avec une page de présentation et une page catalogue;
  • Rendre possible l’ajout futur de nouveaux jeux sans refonte complète du site;
  • Prévoir une progression de difficulté afin de maintenir l’intérêt de l’utilisateur;
  • Produire une base de code propre et ompréhensible.

3 - Public visé

Le public comprend :

  • Les adolescents et adultes recherchant un divertissement intelligent;
  • Les personnes souhaitant exercer leur mémoire ou leur concentration;
  • Les personnes curieuses de suivre leurs performances sur des jeux courts.

4 - Périmètre fonctionnel

La première version du projet couvre les fonctions suivantes :

4.1 - Fonctionalités incluses

  • Page d'accueil : Présentation du projet, de ses objectifs et accès à la plateforme.
  • Catalogue de jeu : Affichage des jeux disponibles avec une courte description et un bouton de lancement.
  • Jeu Memory : Première implémentation complète d’un mini-jeu cognitif.
  • Gestion de la difficulté : adaptation du niveau, par exemple via le nombre de cartes ou la complexité.
  • Architecture modulaire : organisation du projet de sorte qu’un nouveau jeu puisse être ajouté facilement.
  • Identité visuelle cohérente : Respect d’une charte graphique simple, homogène et lisible..

4.2 Fonctionnalités hors périmètre immédiat

Ces éléments sont identifiés comme des pistes d’évolution, mais ne constituent pas des exigences de la première version :

  • Système complet d’inscription et de connexion;
  • Base de données distante;
  • Tableau de bord avancé de progression;
  • Multilingue complet;
  • Grand nombre de jeux dès la première livraison.

5 - Exigences fonctionnelles

Ref. Exigence
EF-01 L’application doit proposer une page d’accueil présentant le projet et permettant d’accéder au catalogue.
EF-02 L’application doit afficher un catalogue des jeux disponibles.
EF-03 L’utilisateur doit pouvoir lancer un jeu depuis le catalogue.
EF-04 Le Memory doit être jouable de bout en bout sans erreur bloquante.
EF-05 La difficulté du jeu doit pouvoir évoluer selon le niveau ou la progression de l’utilisateur.
EF-06 L’architecture doit permettre l’ajout d’un nouveau jeu avec un impact limité sur le reste du projet.
EF-07 L’interface doit rester lisible sur ordinateur et sur mobile.
